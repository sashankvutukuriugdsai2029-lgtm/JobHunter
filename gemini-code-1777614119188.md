# Features

## MVP Feature 1: AI-Assisted Onboarding
**Description:** Ingest a PDF resume and use an LLM to auto-fill the user's profile and initial search strategy.
**Acceptance Criteria:**
- [ ] Streamlit `st.file_uploader` accepts PDFs.
- [ ] Parse PDF text and send to LangChain LLM with `CandidateProfile` and `SearchStrategy` Pydantic output parsers.
- [ ] Render a form pre-filled with the LLM's extracted data.
- [ ] Allow user to manually edit fields (Salary, Location, Visa, Dealbreakers).
- [ ] Save to `JobHunterState` and checkpoint to SQLite.
**Priority:** MVP - Critical

## MVP Feature 2: The Base Graph & Persistence
**Description:** Set up LangGraph state machine so memory is retained.
**Acceptance Criteria:**
- [ ] Initialize `SqliteSaver` checkpointer using a `thread_id`.
- [ ] Create graph routing logic (e.g., if `is_onboarded == False`, route to onboarding node).
**Priority:** MVP - Critical

## MVP Feature 3: Job Fetcher & Dealbreaker Filter
**Description:** Load jobs from a CSV and strictly filter them.
**Acceptance Criteria:**
- [ ] Load `jobs.csv` via Pandas.
- [ ] Filter out roles that do not match `visa_status` or `work_model`.
- [ ] `ranker` node assigns a 0-100 fit score based on skills.
**Priority:** MVP - Critical

## MVP Feature 4: Gmail Integration & Feedback Parsing
**Description:** Pull real data into the app to drive the agent's logic.
**Acceptance Criteria:**
- [ ] Authenticate with Google API to read emails.
- [ ] LLM extracts a 1-sentence reason for rejection from raw email text.
**Priority:** MVP - Critical

## MVP Feature 5: Pattern Detection & Interrupt
**Description:** The "Brain" of the app. Detect trends and ask for permission to pivot.
**Acceptance Criteria:**
- [ ] Trigger only if `len(feedback_log) >= 3`.
- [ ] Use LangGraph `interrupt_before=["strategy_updater"]`.
- [ ] UI intercepts paused graph and renders "Approve/Reject".
**Priority:** MVP - Critical
