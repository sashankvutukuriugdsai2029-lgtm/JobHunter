# Tech Stack

## Core Logic & Agent Framework
**Framework: LangGraph (Latest Stable)**
- Rationale: Required for long-running, cyclical state machine architecture and human-in-the-loop interrupts.
**State Validation: Pydantic v2**
- Rationale: Non-negotiable for strictly typing the graph state. 
**LLM Orchestration: LangChain**
- Rationale: Connects seamlessly with OpenAI/Claude/Gemini.

## Data Ingestion & Processing
**PDF Parsing: PyPDF2 or LangChain PyPDFLoader**
- Rationale: Required for the "Magic Onboarding" step to convert the uploaded CV into raw text for the LLM to extract profile data.

## Persistence & Storage
**Agent Memory: SQLite via LangGraph `SqliteSaver`**
- Rationale: Provides an offline-first, reliable checkpointer. Uses a unique `thread_id` tied to the user.
**Job Database: Local CSV (Pandas)**
- Rationale: Eliminates the risk of live-scraping failures during a high-stakes demo. 

## Frontend UI
**Framework: Streamlit**
- Rationale: Fastest way to build a 3-panel reactive dashboard with native file-upload and session state components.

## Integrations
**Email Scanning: Google Gmail API (OAuth 2.0)**
- Rationale: Professional integration to automatically fetch rejection emails.
