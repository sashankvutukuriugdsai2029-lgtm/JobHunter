# Business Logic & Rules Engine

## Onboarding Fallbacks
If the CV parsing LLM cannot determine the candidate's seniority from their resume text, it must default `target_seniority` to "Mid-Level". If it cannot determine target roles, it defaults to "Software Engineer".

## Strict Filtering Rules
Before ranking jobs, the `job_fetcher` must execute "Hard Filters":
1. **Visa:** If user `visa_status` == "Requires Sponsorship", instantly discard any CSV job where `sponsorship_available == False`.
2. **Work Model:** If user `work_model` == "Remote", instantly discard any job marked "On-site" or "Hybrid".
3. **Salary:** Discard any job where the maximum salary/stipend is lower than the user's `target_salary`.

## Pattern Trigger Thresholds
- **Rule 1:** The `pattern_detector` may only propose a pivot if it finds a minimum of **3** occurrences of the *exact same underlying reason* in the log.
- **Rule 2:** Once a strategy is updated based on a pattern, the agent cannot suggest reverting that specific change for at least 1 "Session cycle".
